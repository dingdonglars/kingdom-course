# Quiz — Module 3.0 (reading-themed)

## 1. Why read other people's code before writing your own?

a. To copy it
b. Reading builds *judgement* — knowing why one structure is better than another. Tutorials teach syntax; reading teaches taste.
c. Required by the curriculum
d. To pad the schedule

## 2. What's the right ratio of reading to writing for a senior?

a. ~5x more reading than writing — they scan, find the right file, notice missing things
b. 1:1
c. Writing only
d. Reading only

## 3. The "five questions" mini-protocol asks about *one thing that surprises you*. Why?

a. To pad the journal
b. The surprise is the gap between what you assumed and what's actually there — exactly the spot where new understanding lands
c. Pure habit
d. Required by xUnit

## 4. Why does the lesson say "don't Google answers as you read"?

a. Anti-Google policy
b. Trying first, then checking, builds judgement. If you Google immediately, you import understanding without ever stretching.
c. Internet might be slow
d. To save bandwidth

## 5. Reading, writing-down notes, and committing them — why is the *commit* part important?

a. Reading-and-noting is real work; making it visible in your git log treats it like any other task. It also creates a paper trail.
b. To pass the M4 challenge
c. Optional decoration
d. Required by GitHub
