# Module 3.5 — Google Cloud Console setup notes

> Private notes from setting up the OAuth client. **Don't commit the actual ClientId/Secret values** — those are in user-secrets.

## Steps taken
- [ ] Created project: ____________________
- [ ] Enabled OAuth consent screen (External, "Kingdom" app name)
- [ ] Created OAuth Web application credentials
- [ ] Added authorized redirect URI: `https://localhost:7XXX/signin-google`
- [ ] Copied ClientId + ClientSecret immediately
- [ ] Stored both in user-secrets:
  - `dotnet user-secrets set "Google:ClientId" "..."`
  - `dotnet user-secrets set "Google:ClientSecret" "..."`

## When deploying (Module 3.8)
- [ ] Add prod redirect URI: `https://kingdom-api.azurewebsites.net/signin-google` (or your domain)
- [ ] Set env vars in App Service: `Google__ClientId`, `Google__ClientSecret` (note: double underscore)
