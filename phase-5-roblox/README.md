# Phase 5 — Roblox Kingdom

> **Your game lives where your friends already play.**

**What you'll have at the end:** A version of your Kingdom that your friends play on Roblox. The engine — the same one you've been carrying since Phase 1, now ported four times across console, file, web API, and browser — survives one more port: from C# into Luau. The point of the whole year lands in this phase, in your hands.

**Estimated effort:** 8 weeks at 4–6 hrs/week  ·  **Wraps:** **M6** *Kingdom v5 — Roblox-Published*

---

## Why this phase?

Look at where you started. Phase 0 was a program that printed roasts. Phase 1 added classes and a game loop. Phase 2 saved the world to disk, then to a database. Phase 3 put it on the web. Phase 4 put it in a browser. Five times now you've taken the same idea — *the engine knows what to do; the shell decides when* — and proven it in a different runtime. Phase 5 proves it one more time, in a different language entirely, on the platform where most of your friends spend their evenings.

Roblox is also where the audience already is. The previous phases earned you a working game that you could show someone if they sat next to you. Phase 5 ends with a URL you send. They click it. They play. That's the difference this phase is designed to land.

The phase is also a wrap-up. You'll feel it in the writing — the tone gets a little quieter, the lessons reach back to earlier modules and tie threads together. M5.8 is the last numbered module of the year.

## What you'll learn (named)

- Roblox Studio — the editor, the panels, how a game project is organised
- Luau — Lua with optional types; the syntax bridge from C# and JavaScript
- OOP without classes — tables and metatables, the Lua way
- Server vs client, RemoteEvents, replication — the multiplayer-first runtime
- Porting an engine across languages without losing its meaning
- Spawning a 3D world from code
- DataStore — Roblox's built-in persistence
- Publishing a place so anyone with a link can play

## What you'll build (the climb)

- **Week 41:** install Studio; tour the editor; run your first script.
- **Week 42:** Luau basics — variables, functions, conditionals, loops, tables.
- **Week 43:** classes via metatables; the Phase 1 engine starts moving across.
- **Week 44:** server vs client, RemoteEvents, the rules of multiplayer.
- **Weeks 45–46:** the engine reaches feature parity in Luau; tiles spawn from code.
- **Week 47:** DataStore persistence — the kingdom survives across sessions.
- **Week 48:** publish the place. Send the URL. M6.

## Brag-worthy outcome

A public Roblox game your friends play, on Roblox, with your engine, written by you — across two languages, two runtimes, and one model that didn't change.
