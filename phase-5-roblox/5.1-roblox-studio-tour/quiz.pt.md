## 1. Onde ficam os scripts do lado do servidor no Roblox?

- **a.** `Workspace`
- **b.** `ServerScriptService`

- **d.** `StarterPack`

## 2. Para que serve o painel Output?

- **a.** Mostrar o progresso de build de um place
- **b.** O console — chamadas de `print(...)` e erros de tempo de execução aparecem aqui
- **c.** Mixagem de áudio para os sons do place
- **d.** Uma segunda visão do viewport 3D

## 3. Os quatro painéis principais do Roblox Studio são...

- **a.** Code, Build, Run, Debug
- **b.** Viewport, Explorer, Properties, Output
- **c.** Scene, Hierarchy, Inspector, Console
- **d.** Files, Edit, Run, Help

## 4. O que é um *Place* no Roblox?

- **a.** Uma localização do mundo real guardada nos metadados do jogo
- **b.** Um mundo do Roblox — o que é publicado e ganha uma URL
- **c.** Uma conta de usuário na plataforma Roblox
- **d.** Uma pasta no `ReplicatedStorage`

## 5. A lição diz para não se assustar com a interface visual do Studio. Por quê?

- **a.** A interface é, na verdade, bem pequena depois que você ignora a barra de ferramentas
- **b.** O trabalho é o mesmo ciclo que você já conhece — achar o arquivo, editar, rodar, ler a saída. O Viewport 3D é um painel; o resto é reconhecível.
- **c.** É uma tradição do Roblox que o editor pareça mais cheio do que é
- **d.** O Roblox exige que a interface visual esteja presente o tempo todo
