## 1. O que é o SSMS?

- **a.** Uma segunda engine de SQL Server, mais leve que a principal
- **b.** SQL Server Management Studio — a interface gráfica oficial do Windows para o SQL Server
- **c.** Um utilitário só de backup que vem com o instalador do LocalDB
- **d.** Um serviço obrigatório que precisa estar rodando para o SQL Server funcionar

## 2. A alternativa multiplataforma ao SSMS é...

- **a.** Não existe uma; você tem que usar uma VM Windows no macOS ou Linux
- **b.** Azure Data Studio — mesmo trabalho, interface um pouco diferente, roda em todo lugar
- **c.** SQLite Browser, readaptado para conexões com o SQL Server
- **d.** Necessário para usar a imagem Docker do SQL Server

## 3. O que o Plano de Execução Estimado te diz?

- **a.** Como o banco de dados planeja rodar uma consulta — buscas por índice, varreduras, joins, contagens de linha
- **b.** Se uma consulta atende a um padrão de desempenho definido pela Microsoft
- **c.** O tempo de execução em milissegundos, na média da última hora
- **d.** Um passo obrigatório antes de qualquer consulta poder rodar pela primeira vez

## 4. Por que "conhecer bem uma ferramenta profissional" vale o tempo?

- **a.** Ferramentas profissionais transformam minutos de trabalho em segundos, e o tempo economizado se acumula ao longo de uma carreira
- **b.** O SSMS é a única forma legal de consultar um SQL Server em produção
- **c.** A Microsoft exige certificação em SSMS antes de você poder publicar código
- **d.** Martelos e outras ferramentas manuais são ruins em geral

## 5. O que **clicar com o botão direito no banco de dados → Generate Scripts** produz?

- **a.** Um gráfico de desempenho do banco de dados na última semana
- **b.** Um arquivo `.sql` contendo o esquema, opcionalmente com os dados — útil para backups e para compartilhar
- **c.** Um hash do banco de dados para verificação de integridade
- **d.** Uma saída obrigatória durante a instalação; não para uso normal
