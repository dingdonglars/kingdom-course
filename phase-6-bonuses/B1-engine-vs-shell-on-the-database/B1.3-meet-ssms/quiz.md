## 1. What is SSMS?

- **a.** A second SQL Server engine, lighter than the main one
- **b.** SQL Server Management Studio — the official Windows GUI for SQL Server
- **c.** A backup-only utility that ships with the LocalDB installer
- **d.** A required service that has to be running for SQL Server to work

## 2. The cross-platform alternative to SSMS is...

- **a.** There isn't one; you have to use a Windows VM on macOS or Linux
- **b.** Azure Data Studio — same job, slightly different UI, runs everywhere
- **c.** SQLite Browser, repurposed for SQL Server connections
- **d.** Required to use the SQL Server Docker image

## 3. What does the Estimated Execution Plan tell you?

- **a.** How the database plans to run a query — index seeks, scans, joins, row counts
- **b.** Whether a query meets a performance benchmark set by Microsoft
- **c.** The execution time in milliseconds, averaged over the last hour
- **d.** A required step before any query can run for the first time

## 4. Why is "knowing one professional tool well" worth the time?

- **a.** Professional tools turn minutes of work into seconds, and the time saved adds up over a career
- **b.** SSMS is the only legal way to query a SQL Server in production
- **c.** Microsoft requires SSMS certification before you can ship code
- **d.** Hammers and other manual tools are bad in general

## 5. What does **right-click database → Generate Scripts** produce?

- **a.** A performance graph of the database over the last week
- **b.** A `.sql` file containing the schema, optionally with the data — useful for backups and sharing
- **c.** A hash of the database for integrity checking
- **d.** Required output during install; not for regular use
