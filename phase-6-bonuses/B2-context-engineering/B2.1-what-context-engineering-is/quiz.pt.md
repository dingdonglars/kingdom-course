## 1. O modelo de quatro passos para uma interação com IA é...

- **a.** Prompt → contexto → saída → avaliação
- **b.** Pergunta → resposta → revisão → pronto
- **c.** Código → teste → entregar → esquecer
- **d.** Planejar → escrever → revisar → entregar

## 2. Quais três modos de falha respondem por quase toda saída ruim de IA?

- **a.** Saída de tutorial genérico, APIs inventadas, desvio do estilo
- **b.** Geração lenta, tokens caros, dados de treino enviesados
- **c.** Linguagem errada, caminho de arquivo errado, versão de sintaxe errada
- **d.** Nenhum — a IA moderna é confiável o bastante para você aceitar os padrões

## 3. A lição diz "você não controla a IA; você controla o contexto dela." O que isso quer dizer na prática?

- **a.** O passo de geração em si é opaco, mas o prompt, o scaffolding, o scoping e a sua avaliação são todos seus — então gaste esforço aí
- **b.** A saída da IA é fundamentalmente quebrada e deve ser tratada com desconfiança o tempo todo
- **c.** Desempenho é a preocupação principal; a engenharia de contexto otimiza para velocidade
- **d.** Tradição — desenvolvedores mais velhos preferem assim

## 4. Qual é a diferença entre scaffolding e scoping?

- **a.** São a mesma coisa com dois nomes diferentes
- **b.** Scaffolding é o pano de fundo persistente (ARCHITECTURE, STANDARDS, exemplos). Scoping é o enquadramento por tarefa (objetivo, armadilhas, indicação de estilo para *este* pedido).
- **c.** Scoping é mais rápido de montar que scaffolding
- **d.** Scaffolding é exigido pelo Claude; scoping é opcional

## 5. O que é o passo de avaliação (eval)?

- **a.** O seu julgamento sobre se a saída está certa *para o seu projeto* — a IA não conhece os seus padrões, você conhece
- **b.** Um padrão de desempenho rodado depois de cada resposta da IA
- **c.** Um framework de testes que vem com a maioria das ferramentas de IA
- **d.** A autoavaliação da própria IA sobre a saída dela
