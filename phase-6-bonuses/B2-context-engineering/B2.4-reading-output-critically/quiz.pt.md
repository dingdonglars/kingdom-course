## 1. A leitura de cinco coisas para a saída de código de uma IA é...

- **a.** APIs inventadas, caminho infeliz, desvio de estilo, ajuda demais, erros engolidos
- **b.** Bugs, desempenho, estilo, testes, documentação
- **c.** Compilar, rodar, testar, entregar, implantar
- **d.** Ler, entender, aceitar, fazer merge, esquecer

## 2. O que é uma API inventada?

- **a.** Um novo recurso adicionado ao runtime do .NET numa versão recente
- **b.** Uma chamada de método que a IA gera com confiança — mas o método não existe no seu código nem nas suas bibliotecas
- **c.** Um erro de digitação no nome de um método, que o compilador vai pegar no próximo build
- **d.** Uma incompatibilidade de versão de C# entre os dados de treino da IA e o seu projeto

## 3. Por que `catch { return null; }` é um sinal de alerta em código gerado por IA?

- **a.** Ele engole erros reais silenciosamente — uma falha de rede vira "nenhum reino" em vez de uma exceção sobre a qual você pode agir
- **b.** Ele roda mais devagar que capturar um tipo específico de exceção
- **c.** O runtime do .NET exige que ele seja removido
- **d.** Preferência de estilo; nada de fato errado com ele

## 4. Por que ajuda demais é um sinal amarelo numa resposta de IA?

- **a.** A IA adicionou um comportamento que você não pediu — ou está errado (desfazer desperdiça tempo) ou é uma decisão não autorizada (expansão de escopo)
- **b.** Ajuda demais deixa a IA mais lenta e aumenta o custo de tokens
- **c.** Exigido pelo treino da IA, que adiciona recursos extras por padrão
- **d.** IAs prestativas são ruins em geral; saída mínima é sempre melhor

## 5. O padrão da lição não é "perfeito" — é "saída que eu consigo defender na viva". O que isso quer dizer?

- **a.** Você vai ter que explicar cada linha. Código que você consegue explicar é entregue; código que você não consegue, não. A viva é o teste prático de avaliação.
- **b.** A IA é arriscada demais para usar em projetos reais; só código avaliado em viva conta
- **c.** Exigido pelo currículo, mas não pela indústria em geral
- **d.** Preferência de estilo; "defender" é só uma palavra de que a lição gosta
