# GLOSSARY.md

> Living alphabetical glossary of every term introduced in any lesson. Each entry is one line: a definition the learner can read in plain English, and the lesson where the term first appears.

> **How this list grows:** every lesson with a "Words to watch" sidebar adds its terms here at authoring time. `/lesson-review` and the final consistency audit catch missing entries.

## A

- **argument** ‚Äî the actual value you pass when calling a method (the *parameter* is the name in the definition; the *argument* is the value). *Module 0.6.*
- **ASCII art** ‚Äî pictures made out of text characters. *Module 0.4.*

## B

- **`bool`** ‚Äî `true` or `false`, nothing else. *Module 0.5.*
- **breakpoint** ‚Äî a marked line where the debugger pauses execution. *Module 0.8.*

## C

- **call stack** ‚Äî the chain of methods that called each other to reach the current point of execution. *Module 0.8.*
- **camelCase** ‚Äî convention for local variables and parameters (`goldCount`, `firstName`). *Module 0.5.*
- **`_camelCase`** ‚Äî convention for private fields inside classes (one underscore + camelCase). *Module 0.5.*
- **cast** ‚Äî convert a value from one type to another (`(int)3.99` is `3`). *Module 0.5.*
- **collection** ‚Äî any data structure that holds multiple values (list, dictionary, array). *Module 0.7.*
- **conditional** ‚Äî `if` / `else` ‚Äî a fork in the code. *Module 0.2.*
- **`Console.ForegroundColor`** ‚Äî the property that controls the color of text written to the console from now on. *Module 0.4.*
- **`Console.ReadLine()`** ‚Äî the method that pauses, waits for input, returns the line as a string. *Module 0.1.*

## D

- **`DateTime`** ‚Äî a date *and* a time, together. *Module 0.5.*
- **debugger** ‚Äî a tool that pauses your program at chosen points so you can see what's happening. *Module 0.8.*
- **`Dictionary<K, V>`** ‚Äî a lookup table from `K` (key) to `V` (value); each key appears at most once. *Module 0.7.*
- **`double`** ‚Äî a number that can have a decimal point. *Module 0.5.*

## E

- **exception** ‚Äî a runtime error that the program couldn't handle and threw. *Module 0.8.*

## F

- **`File.ReadAllText`** ‚Äî reads a file's entire contents as a string. *Module 0.4.*
- **`File.WriteAllText`** ‚Äî writes a string to a file (creates if missing; overwrites if present). *Module 0.4.*
- **`for`** ‚Äî counter-based loop (`for (int i = 0; i < n; i++)`). *Module 0.7.*
- **`foreach`** ‚Äî loops through every item in a collection, one at a time. *Module 0.7.*

## I

- **`IInterface`** ‚Äî convention for interface names (`IBuilding`, `IRandom`). *Module 0.5.*
- **`int`** ‚Äî a whole number (no decimal point); range about ¬±2 billion. *Module 0.5.*
- **`int.Parse(...)`** ‚Äî turns a string of digits into an `int`; throws if not a valid integer. *Module 0.2.*

## L

- **LINQ** ‚Äî Language-Integrated Query: methods like `.Where`, `.Select`, `.OrderBy`, `.Sum` that work on any collection. *Module 0.7.*
- **list** ‚Äî an ordered collection of things you can add to, remove from, and loop through. *Module 0.3.*
- **`List<string>`** ‚Äî a list specifically of strings. The `<string>` says "this is a list of strings"; you'll see `<int>`, `<Building>` later. *Module 0.3.*
- **`List<T>`** ‚Äî generic ordered list of `T`s. *Module 0.7.*
- **loop** ‚Äî a chunk of code that runs repeatedly until told to stop. *Module 0.2.*

## M

- **method** ‚Äî a named chunk of code that does one thing. *Module 0.1 (called); Module 0.3 (defined); Module 0.6 (deep dive).*

## N

- **nullable** ‚Äî a type that's also allowed to hold `null` (no value at all); written with a trailing `?`. *Module 0.5.*

## O

- **organisation** ‚Äî splitting code into multiple methods so each does one thing. *Module 0.3.*
- **overload** ‚Äî multiple methods with the same name but different parameter types. *Module 0.6.*

## P

- **parameter** ‚Äî the name of an input declared in a method definition (`string kingdom` in `Greet(string kingdom)`). *Module 0.6.*
- **PascalCase** ‚Äî convention for type names, method names, and properties (`Building`, `Console.WriteLine`). *Module 0.5.*

## R

- **README** ‚Äî the doc at the top of every repo. Four sections that matter: what / how to run / what you learned / what's next. *Module 0.4.*
- **return value** ‚Äî the value a method gives back; declared by the type before the method name. *Module 0.6.*

## S

- **`static`** ‚Äî a method or field belonging to the type itself, not to any specific instance. *Module 0.6 (preview); Module 1.1 (depth).*
- **`string`** ‚Äî a piece of text in code, in `"double quotes"`. *Module 0.1; Module 0.5 (formal).*
- **string interpolation** ‚Äî the `$"..."` syntax that lets you stick variables inside a string with `{curly braces}`. *Module 0.1.*

## T

- **`throw`** ‚Äî the keyword for raising an exception. *Module 0.8.*
- **`try / catch`** ‚Äî wrap risky code in `try`; if it throws, the matching `catch` block runs. *Module 0.8.*
- **type** ‚Äî what kind of value something is (number, text, true/false, date). *Module 0.5.*

## V

- **variable** ‚Äî a named place to store a piece of data so you can use it later. *Module 0.1.*
- **`void`** ‚Äî the special return type meaning "this method gives nothing back". *Module 0.6.*

## W

- **`while`** ‚Äî keyword starting a "keep doing this while the condition is true" loop. *Module 0.2.*
