# Kingdom — A Programming Course

> Welcome. This repo holds a self-guided programming course built around one mainstay project: **the Kingdom**, a tycoon-style management game that grows from a console application all the way to a Roblox world. Across about a year, you'll build it five times — same engine, different runtimes — and learn what every developer needs to know along the way.

## Who this is for

You are 15ish. You've never written code. You play games. You have a Windows machine and ~4–6 hours a week. You're going to ship things your friends can play.

## What you'll build

| Block | What you ship |
|---|---|
| 1 — Spark Week | Four playable toys in a month |
| 2 — Foundations | A small CLI tool you'll actually use |
| 3 — Console Kingdom | Your own tycoon game running in a terminal |
| 4 — Persistence | Your kingdom survives a restart |
| 5 — Web API on the Internet | Friends play your game from anywhere |
| 6 — Browser Kingdom | Your game runs in any browser, no install |
| 7 — Roblox Kingdom *(bonus)* | Your game lives where your friends already play |
| 8 — Bonuses *(bonus)* | Capstone bonuses (DB engine + AI-driven dev) |

By the end of Block 5 (~month 12), the public version of your game is on the internet with real OAuth login and a leaderboard. The Roblox version trails into the months after.

## How the course is laid out

- **Blocks** are the *advertised stop points*. Each one ends on a brag-worthy artefact. Each has its own opener (a `README.md`) that pitches the block.
- **Phases** are the *engineering structure* underneath. Each phase has modules, each module has a lesson. Lessons follow the same six-beat shape (hook → do it → tinker → name it → quiz → connect).
- **Milestones** (M0–M6) are the wrappers — your milestone PRs are the artefacts you and Lars review together.

## Where to start

1. Read `for-you.md` — the AI-tools guide.
2. Read `STYLE.md` — what every lesson looks like.
3. Read `STANDARDS.md` — the code, naming, and PR conventions.
4. Read `ENGLISH-NOTES.md` — why the course is in English and what scaffolding exists.
5. Read `phase-0-spark/README.md` — your first pitch (Block 1).
6. Open `phase-0-spark/0.0-setup-and-roast-o-matic/lesson.md` — start here.

## Where to ask for help

- **Lars** — your mentor. Channels and rules are in the *Mentor Protocol* (Lars gives you that document outside this repo).
- **Claude** — your AI assistant. Read `for-you.md` first to understand what to ask for and what not to.

## A note on AI

This course teaches programming *with* AI, not against it. There's a moment about halfway through (the **AI Unlock Gate**) when the rules expand. Until then, AI is for friction; after, AI is also for implementation help — under one hard rule. `for-you.md` is the whole story.

## What's NOT in this repo

- The reference Kingdom (a working version of the project, written by Lars). Lives in a separate public repo. **You should NOT open it until your phase is done** — see the "Compare, Don't Conform" protocol in your milestone reviews.
- Your own work — that lives in *your* repo, which you'll create on Module 0.0 from `starter-template/`.
