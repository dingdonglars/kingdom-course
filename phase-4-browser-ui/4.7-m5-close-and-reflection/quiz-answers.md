# Quiz answers — Module 4.7

## 1. a
Re-reading old code with current eyes is the cheapest, highest-signal mastery test there is. The growth is concrete: you can name what you'd change and why. That's not a feeling — it's evidence.

## 2. b
Generic dissatisfaction is anxiety: "this code is bad somehow." It doesn't translate to action. Specific dissatisfaction is mastery: "I'd extract this method because it does three things; I'd rename `data` to `kingdomSnapshot`; this comment is no longer accurate." Each item is a discrete action.

## 3. a
Refactoring all of Phase 0 is busywork (and the toys aren't going to be reused). The point is *seeing the gap*. One refactor commits you to articulating it. Doing more doesn't sharpen the lesson; it dulls it.

## 4. a
The Svelte taster is one component — small enough to try without commitment. After it, you have a real opinion: vanilla / Svelte / look at React. The point is informed choice, not a forced switch.

## 5. b
Standard milestone tag pattern: `git tag <name> && git push origin <name>`. Tags are immutable references — they don't move with new commits. The mentor PR points at the tagged state.
