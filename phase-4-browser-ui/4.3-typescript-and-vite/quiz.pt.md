## 1. O que o TypeScript adiciona ao JavaScript?

- **a.** Uma engine de runtime mais rápida que vem com os navegadores modernos
- **b.** Verificação estática de tipos — o mesmo JavaScript roda por baixo, mas o compilador recusa bugs de tipo óbvios
- **c.** Menor uso de memória durante a execução nos navegadores modernos
- **d.** Uma substituição completa do JavaScript, com novas palavras-chave e sintaxe

## 2. O que o Vite faz?

- **a.** Um verificador de tipos para JavaScript que roda no editor
- **b.** Servidor de desenvolvimento e empacotador de frontend moderno — HMR rápido, TypeScript pronto para usar, quase nenhuma configuração
- **c.** Um framework de testes para código de navegador, parecido com o Vitest
- **d.** Um framework de backend para servir APIs a partir de projetos JavaScript

## 3. O que é HMR?

- **a.** Hard Memory Reset — limpa o cache do navegador entre execuções
- **b.** Hot Module Replacement — edite, salve, e o navegador atualiza sem perder o estado da página
- **c.** Um modo de depuração para rastrear bugs de JavaScript difíceis de reproduzir
- **d.** Uma configuração obrigatória em configurações de HTTPS em frontends modernos

## 4. O cast `as KingdomSlot[]` no JSON interpretado — por que ele é necessário?

- **a.** Exigido pela interpretação de JSON nos navegadores modernos e nos runtimes Node
- **b.** O compilador não consegue verificar o layout da API em tempo de execução; `as` é a sua afirmação de que o objeto interpretado combina com a interface
- **c.** Ele acelera o passo de interpretação de forma mensurável para corpos de resposta grandes
- **d.** Uma preferência de estilo; o código funciona igual sem ele

## 5. Por que a regra é "tipos em toda fronteira"?

- **a.** Fronteiras são onde os erros de layout acontecem — formato de rede, formato de arquivo, contrato de API — e os tipos deixam o contrato explícito
- **b.** Exigido pelo compilador TypeScript antes de ele produzir a saída
- **c.** Mecanismos de busca agora recompensam sites com disciplina de tipos mais rígida nas fronteiras
- **d.** Execução mais rápida em tempo de execução quando os tipos da fronteira são declarados primeiro
