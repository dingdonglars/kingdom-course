# Quiz — Module 4.1

## 1. What's the difference between HTML and CSS?

a. They're identical
b. HTML describes the *structure* (sections, headings, lists); CSS describes the *appearance* (colors, spacing, fonts)
c. HTML is for input, CSS for output
d. HTML is server-side, CSS is client-side

## 2. Why use semantic elements (`<header>`, `<main>`, `<nav>`) instead of `<div>` everywhere?

a. They're shorter to type
b. Meaningful tags improve accessibility (screen readers), SEO, and code readability
c. Required by HTML5
d. They're styled by default

## 3. What does `box-sizing: border-box` do?

a. Adds a border
b. Makes element dimensions include padding + border (sane width math); without it, padding adds to the declared width
c. Removes the box
d. Required by CSS

## 4. Why prefer system fonts (`system-ui`)?

a. Native to each OS, fast (no download), feels right by default
b. Required by browsers
c. Tradition
d. Only Linux supports them

## 5. The lesson says "markup first, style second, behaviour third." Why?

a. Build semantic HTML that works with no CSS or JS; *then* style it; *then* add behaviour. Skipping steps creates mess.
b. Required by web standards
c. Faster
d. SEO requires it
