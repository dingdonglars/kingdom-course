## 1. Qual é a diferença entre HTML e CSS?

- **a.** Fazem a mesma coisa; os nomes são intercambiáveis
- **b.** HTML descreve a estrutura (seções, títulos, listas); CSS descreve a aparência (cores, espaçamento, fontes)
- **c.** HTML cuida da entrada; CSS cuida da saída da página
- **d.** HTML roda no servidor; CSS roda só no navegador

## 2. Por que usar elementos semânticos (`<header>`, `<main>`, `<nav>`) em vez de `<div>` em todo lugar?

- **a.** Eles levam menos caracteres para digitar do que um `<div>` com uma classe
- **b.** Tags com significado ajudam leitores de tela, mecanismos de busca e a legibilidade do código
- **c.** São uma exigência rígida da especificação HTML5
- **d.** Os navegadores os renderizam com estilos embutidos que você não pode mudar

## 3. O que `box-sizing: border-box` faz?

- **a.** Adiciona uma borda fina em volta de todo elemento da página
- **b.** Faz a largura do elemento incluir o preenchimento (padding) e a borda, então `width: 200px` ocupa de fato 200 pixels
- **c.** Remove o layout de caixa por completo e usa renderização em linha
- **d.** Exigido por todo arquivo CSS antes de as outras regras valerem

## 4. Por que preferir `font-family: system-ui`?

- **a.** Nativa de cada sistema operacional, sem download, fica com a cara certa por padrão em todo lugar
- **b.** É o único valor de fonte que o interpretador de CSS aceita
- **c.** Convenção mais antiga; projetos modernos usam por motivos de compatibilidade
- **d.** Só o Linux suporta fontes personalizadas, então essa é a alternativa para todos os sistemas

## 5. A lição diz "marcação primeiro, estilo segundo, comportamento terceiro." Por quê?

- **a.** Construa HTML semântico que funcione sem CSS nem JS, depois estilize, depois adicione interatividade — pular passos cria uma bagunça
- **b.** É uma regra rígida do órgão de padrões web W3C
- **c.** É mensuravelmente mais rápido escrever código nessa ordem
- **d.** Mecanismos de busca se recusam a indexar sites que não seguem a ordem
