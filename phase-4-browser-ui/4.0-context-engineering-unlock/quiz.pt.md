> Tente primeiro o `quiz.md` em inglês. Use este aqui só quando uma palavra te travar.

## 1. O que é *engenharia de contexto*?

- **a.** Escolher o que a IA vê antes de ela responder, para a saída dela combinar com o seu projeto
- **b.** Um padrão C# para gerenciar o escopo da injeção de dependência
- **c.** Uma biblioteca de log usada em frameworks web modernos
- **d.** Um recurso embutido em todo produto de chat de IA

## 2. O modelo de quatro passos para pedidos de implementação a uma IA é...

- **a.** Objetivo · onde isso vai · convenções · o que *não* fazer
- **b.** Título · descrição · autor · data
- **c.** Try · catch · finally · throw
- **d.** Sujeito · verbo · objeto · tempo

## 3. O que é uma *API inventada*?

- **a.** Um novo endpoint que você adicionou ao seu projeto recentemente
- **b.** Um modo de falha em que a IA chama, com confiança, métodos que não existem na sua base de código
- **c.** Um método não documentado numa biblioteca de terceiros
- **d.** Uma extensão de especificação OpenAPI que você ainda não viu

## 4. Por que organizar um *conjunto de referência* (STANDARDS, ARCHITECTURE, exemplos)?

- **a.** A IA parte das convenções do seu projeto em vez de suposições genéricas de tutorial; a qualidade da resposta é limitada pela qualidade do contexto
- **b.** Exigido pelo .NET SDK antes de os recursos de IA funcionarem
- **c.** Revisores esperam ver um em todo commit que você faz
- **d.** Melhora a velocidade de resposta da IA de forma mensurável

## 5. A regra pós-Unlock diz que você precisa conseguir explicar cada linha que você entrega. Por quê?

- **a.** Código assistido por IA que você não consegue explicar é código que vai apodrecer — quando quebra, você não consegue depurar; quando é revisado, você não consegue defender
- **b.** É exigido pelo compilador C# para arquivos tocados por IA
- **c.** Linters vão sinalizar qualquer código que esteja sem uma explicação
- **d.** O GitHub agora bloqueia PRs sem uma explicação escrita para cada linha
