# Quiz — Module 0.6

## 1. What does `void` mean as a return type?

a. The method takes no parameters
b. The method gives nothing back
c. The method is undefined
d. The method runs in the background

## 2. Given `int Square(int x) => x * x;`, what is `Square(7)`?

a. `7`
b. `14`
c. `49`
d. An error

## 3. Can you have two static methods both called `Square` in the same class?

a. No — names must be unique
b. Yes, if they take different parameter types (different overloads)
c. Yes, if they take the same parameters but have different return types
d. Yes, if their parameter names are different

## 4. What's the difference between a *parameter* and an *argument*?

a. They mean the same thing
b. Parameter = the name in the method definition; argument = the actual value passed when calling
c. Parameter = the value passed; argument = the name in the definition
d. Parameter = required, argument = optional

## 5. `int AddGold(int a, int b) => a + b;` and `int AddGold(int a, int b) { return a + b; }` — are these the same?

a. Yes — the `=>` syntax is shorthand for `{ return ...; }`
b. No — the first one runs faster
c. No — the first is "expression-bodied"; the second is "block-bodied"; they behave the same way
d. Both (a) and (c) are correct
