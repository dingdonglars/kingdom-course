# Block 2 — Foundations

> **Real C# muscle. Build a tool you'll actually use.**

**What you'll have at the end:** An Inventory Tool — a small command-line program that manages a list of things (add / remove / find / save). Plus a deeper understanding of the language pieces you've already been using.

**Estimated effort:** 2 weeks at 4–6 hrs/week  ·  **Wraps:** M1 *Inventory Tool*

---

## Why this block?

You shipped four toys in Block 1 without understanding the language formally. That's *good* — you have the experience. Now we put names on what you used: types, methods, collections, errors. After this block, you can read other people's C# code and roughly know what's happening — which makes the rest of the course move much faster.

The Inventory Tool is the milestone artefact: small enough to finish, real enough to use, and structured enough that you'll genuinely benefit from the names we just gave you.

## What you'll learn (named)

- Types: `int`, `double`, `string`, `bool`, `DateTime`, casting, nullables
- Naming conventions (PascalCase / camelCase / `_camelCase` / `IInterface`) and *why rules exist*
- Methods deeply: parameters, return values, overloads, `static`
- Collections: `List<T>`, `Dictionary<K,V>`, `foreach`, `for`, basic LINQ
- Errors: `try/catch`, the VS Code debugger, breakpoints, the call stack

## What you'll build (the climb)

- **Week 5:** types deep dive (with conventions aside), then methods deep dive.
- **Week 6:** collections, then errors and the debugger. Ship the Inventory Tool at week's end.

## Brag-worthy outcome

A small CLI you'll actually keep on your machine — track your books, your Roblox usernames, your snack inventory, whatever. *Plus* you can now read most beginner C# tutorials without getting lost.
