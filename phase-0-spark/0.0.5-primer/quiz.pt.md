## 1. Você está trabalhando em `C:\code\kingdom\` e o prompt do terminal mostra isso. Você digita `cd ..`. Onde o terminal vai parar?

- **a.** `C:\code\kingdom\..` (uma pasta oculta)
- **b.** `C:\code\` (um nível acima)
- **c.** `C:\` (a raiz do drive)
- **d.** Em lugar nenhum — `..` não faz nada

## 2. Você inicia um programa com `dotnet run`. Ele está imprimindo a saída e o seu prompt sumiu. Você quer pará-lo de forma limpa. O que você faz?

- **a.** Fecha a janela do Windows Terminal no botão X
- **b.** Aperta `Ctrl + C` e espera o prompt voltar
- **c.** Aperta `Esc` várias vezes
- **d.** Abre o Gerenciador de Tarefas e força o encerramento do PowerShell

## 3. Qual é a diferença entre *salvar* um arquivo `.cs` e *executá-lo*?

- **a.** Salvar executa o arquivo; executar salva. Dois nomes para a mesma ação.
- **b.** Salvar grava o código no disco; executar inicia um processo. Ações independentes.
- **c.** Salvar compila o arquivo; executar o envia para o GitHub.
- **d.** Salvar é automático no VS Code; executar tem que ser feito manualmente no Word.

## 4. O Explorador de Arquivos mostra um arquivo chamado `Program`. Qual é o nome de arquivo real dele no disco?

- **a.** Só `Program` — o Windows permite arquivos sem extensão
- **b.** Provavelmente `Program.cs` ou `Program.exe` — o Windows esconde a extensão por padrão, a menos que você ative isso
- **c.** `Program.txt` — o Windows trata todo arquivo desconhecido como texto
- **d.** É uma pasta, não um arquivo

## 5. Um *processo* é:

- **a.** Um arquivo no disco
- **b.** Uma pasta contendo um programa
- **c.** Um programa que está rodando agora no seu computador
- **d.** A mesma coisa que um shell

## 6. O que é o .NET SDK?

- **a.** Um editor de código onde você escreve seus arquivos C#
- **b.** Um conjunto de ferramentas que compila e executa programas C#; o comando `dotnet` é como você conversa com ele
- **c.** Um sistema de controle de versão, como o git
- **d.** Um serviço de hospedagem na nuvem onde aplicações C# ficam
