## 1. You're working in `C:\code\kingdom\` and your terminal prompt shows that. You type `cd ..`. Where does the terminal end up?

- **a.** `C:\code\kingdom\..` (a hidden folder)
- **b.** `C:\code\` (one level up)
- **c.** `C:\` (the drive root)
- **d.** Nowhere — `..` does nothing

## 2. You start a program with `dotnet run`. It's printing output and your prompt is gone. You want to stop it cleanly. What do you do?

- **a.** Close the Windows Terminal window with the X button
- **b.** Press `Ctrl + C` and wait for the prompt to come back
- **c.** Press `Esc` repeatedly
- **d.** Open Task Manager and kill PowerShell

## 3. What's the difference between *saving* a `.cs` file and *running* it?

- **a.** Saving runs it; running saves it. Two names for the same move.
- **b.** Saving writes the code to disk; running starts a process. Independent moves.
- **c.** Saving compiles it; running ships it to GitHub.
- **d.** Saving is automatic in VS Code; running has to be done manually in Word.

## 4. File Explorer shows a file called `Program`. What's its actual filename on disk?

- **a.** Just `Program` — Windows allows files without extensions
- **b.** Probably `Program.cs` or `Program.exe` — Windows hides the extension by default unless you turn it on
- **c.** `Program.txt` — Windows defaults all unknown files to text
- **d.** It's a folder, not a file

## 5. A *process* is:

- **a.** A file on disk
- **b.** A folder containing a program
- **c.** A program that's currently alive on your computer
- **d.** The same as a shell

## 6. What is the .NET SDK?

- **a.** A code editor where you write your C# files
- **b.** A toolkit that compiles and runs C# programs; the `dotnet` command is how you talk to it
- **c.** A version-control system, like git
- **d.** A cloud-hosting service where C# applications live
