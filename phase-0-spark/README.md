# Phase 0 — Spark Week + Foundations

> **Ship four playable toys, then learn the language pieces that hold them together.**

**What you'll have at the end:** A repo with four small console apps you wrote (a snarky roast generator, a sarcastic guessing game, a tiny text adventure, and a polished version of one with ASCII art and saved files), plus a small CLI tool — the Inventory Tool — that you can actually use. By month's end you've written nine modules of code, run a real program nine times, and understood *why* it works.

**Estimated effort:** 6 weeks at 4–6 hrs/week  ·  **Wraps:** **M0** (Joke Toolbox, week 4) and **M1** (Inventory Tool, week 6)

---

## Why this phase?

Most programming courses start with theory. *"Here's what a variable is. Here's what a loop is."* By week three you've forgotten why you signed up. We do the opposite: you make stuff in the first hour, and we name the concepts only *after* you've already used them. **Magic first. Theory after.**

Phase 0 has two halves, and they reinforce each other:

**Spark Week (Modules 0.0–0.4).** You ship four toys in four weeks. Silly on purpose — silliness sticks. By the end you've used variables, loops, conditionals, lists, methods, and files, all without anyone explaining what they "are" first. The point is *experience first*: when we name a concept later, you'll already know what it feels like to use it.

**Foundations (Modules 0.5–0.8).** You slow down and we put names on what you've been using. Types, naming conventions, methods properly, collections, errors and the debugger. The artefact at the end is the **Inventory Tool** — a small CLI you'll actually keep on your machine. Track your books, your Roblox usernames, your snacks, whatever. By the time you finish Phase 0, you can read most beginner C# tutorials and roughly know what's happening — which makes the rest of the course move much faster.

## What you'll learn (named)

- That code does what you tell it — it's not magic, you wrote it
- Variables, strings, methods, loops, conditionals, randomness, lists, files
- Types: `int`, `double`, `string`, `bool`, `DateTime`, casting, nullables
- Naming conventions (PascalCase / camelCase / `_camelCase` / `IInterface`) and *why rules exist*
- Methods properly: parameters, return values, overloads, `static`
- Collections: `List<T>`, `Dictionary<K,V>`, `foreach`, `for`, basic LINQ
- Errors: `try`/`catch`, the VS Code debugger, breakpoints, the call stack

## What you'll build (the climb)

- **Week 1:** install your tools, create your first repo, write a roast generator that prints customisable insults to your friends.
- **Week 2:** a sarcastic Number Guess game that says rude things when you guess wrong.
- **Week 3:** a tiny 3-room text adventure (your own setting).
- **Week 4:** polish — ASCII art, color, save your roast list to a file. (M0 lands here.)
- **Week 5:** types deep dive (with naming conventions), then methods deep dive.
- **Week 6:** collections, then errors and the debugger. Ship the Inventory Tool at week's end. (M1 lands here.)

## Brag-worthy outcomes

After Spark Week: a repo with four tiny games + a README you wrote yourself. The kind of thing you can show a friend and say *"I made this."*

After Foundations: a small CLI you'll actually keep on your machine, plus enough vocabulary to read other people's code without getting lost. *That's* the point where programming stops feeling like a foreign country.
