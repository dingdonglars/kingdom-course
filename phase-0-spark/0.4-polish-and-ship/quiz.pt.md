## 1. O que `Console.ForegroundColor = ConsoleColor.Yellow;` faz?

- **a.** Repinta de amarelo o fundo da janela do console inteira até o programa terminar
- **b.** Define como amarelo a cor de todo o texto escrito a partir de agora, até você mudar de novo
- **c.** Destaca o cursor em amarelo mas deixa a cor do texto como está
- **d.** Lança uma exceção, a menos que o driver do terminal suporte amarelo explicitamente

## 2. O que `Console.ResetColor()` faz?

- **a.** Restaura toda a aplicação de console aos padrões de fábrica, incluindo fonte e tamanho
- **b.** Limpa a saída visível do console, como `Console.Clear()` faria
- **c.** Restaura as cores de frente e de fundo aos padrões do terminal
- **d.** Reposiciona o cursor no canto superior esquerdo da janela do console

## 3. O que `File.WriteAllText("hero.txt", name)` faz?

- **a.** Lê o conteúdo de `hero.txt` para a variável `name`, substituindo o que estava lá
- **b.** Escreve o que está em `name` num arquivo chamado `hero.txt`, criando ou sobrescrevendo ele
- **c.** Acrescenta `name` ao final de `hero.txt`, mantendo intacto qualquer conteúdo já existente
- **d.** Lança um erro, a menos que `hero.txt` já exista na pasta atual

## 4. Onde `hero.txt` vai parar no disco?

- **a.** Sempre na pasta Documentos do Windows do usuário, não importa de onde o programa rode
- **b.** No diretório de trabalho atual do programa (normalmente a pasta do projeto quando você dá `dotnet run`)
- **c.** Na área de trabalho do Windows, pronto para o usuário achar e dar dois cliques
- **d.** Num local de sistema oculto que só administradores conseguem ler

## 5. A anatomia de README desta lição tem quatro seções. Qual NÃO é uma delas?

- **a.** O quê — uma frase dizendo o que o projeto é
- **b.** Como rodar — os comandos de verdade que um estranho digitaria
- **c.** Referência detalhada da API de cada método público do projeto
- **d.** O que vem depois — o que você adicionaria se continuasse
