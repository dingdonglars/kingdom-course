## 1. O que `Console.ReadLine()` faz?

- **a.** Lê a próxima linha do código-fonte do seu programa
- **b.** Pausa seu programa, espera o usuário digitar uma linha e devolve o que ele digitou
- **c.** Lê a primeira linha de um arquivo da pasta atual
- **d.** Devolve a última linha de código que foi impressa no console

## 2. Qual é a diferença entre `Console.Write` e `Console.WriteLine`?

- **a.** `Write` é o nome antigo; `WriteLine` é a nova forma preferida
- **b.** `Write` é para entrada; `WriteLine` é para saída
- **c.** `WriteLine` adiciona uma quebra de linha no final; `Write` deixa o cursor na mesma linha
- **d.** Os dois se comportam de forma idêntica; os nomes são intercambiáveis

## 3. O que `var name = Console.ReadLine();` realmente faz?

- **a.** Diz ao C# que o nome do usuário é a palavra literal "name"
- **b.** Declara uma variável chamada `name` e guarda a entrada do usuário nela
- **c.** Renomeia o programa em execução para o que o usuário digitar
- **d.** Quebra o programa, porque `var` precisa de um tipo explícito

## 4. O que o `$` significa em `$"Hey {name}"`?

- **a.** Marca a string como precisando de substituição de variáveis (interpolação de string)
- **b.** Diz ao C# que essa string representa um preço ou uma quantia em dinheiro
- **c.** É uma forma antiga de escapar caracteres especiais dentro da string
- **d.** Faz a string aparecer em negrito quando é impressa

## 5. Depois de `var name = Console.ReadLine();`, a variável `name` contém...

- **a.** Uma cópia de todo o código-fonte do programa
- **b.** A linha de texto que o usuário digitou antes de apertar Enter
- **c.** O número de caracteres da entrada do usuário
- **d.** Um método que o programa pode chamar depois para ler uma entrada
