## 1. O que `while (true) { ... }` faz?

- **a.** Executa o código de dentro exatamente uma vez e depois segue em frente
- **b.** Executa o código de dentro zero vezes, já que `true` é tratado como um caso especial
- **c.** Executa o código de dentro repetidamente, para sempre, até chegar num `break`
- **d.** Executa o código de dentro uma vez para cada Enter que o usuário aperta

## 2. O que `random.Next(1, 101)` te dá?

- **a.** Um número aleatório entre 1 e 100 incluídos (o limite de cima fica de fora)
- **b.** Um número aleatório entre 1 e 101 incluídos (os dois limites entram)
- **c.** Um número aleatório entre 0 e 101 (os limites são só dicas de início)
- **d.** Sempre o número literal 101 (o limite de cima)

## 3. Numa cadeia `if / else if / else`, quais ramos são executados?

- **a.** Todos os ramos cuja condição for verdadeira, em ordem
- **b.** Nenhum deles — a cadeia só verifica; ela não executa
- **c.** Só o primeiro ramo cuja condição for verdadeira, e aí a cadeia para
- **d.** Só o ramo `else`, depois que toda condição é verificada

## 4. O que `int.Parse("42")` retorna?

- **a.** A string `"42"` sem mudança, já que não dá para converter uma string em número
- **b.** O inteiro `42`, pronto para ser usado em contas
- **c.** Um erro, porque `"42"` não está num formato numérico que o C# reconhece
- **d.** O decimal `42.0`, já que o C# trata entrada numérica como ponto flutuante por padrão

## 5. O que a palavra-chave `break` faz dentro do laço?

- **a.** Pausa o programa por um instante antes de continuar o laço
- **b.** Salta para fora do laço por completo, então o código depois do laço é o próximo a rodar
- **c.** Pula o resto da iteração atual mas continua dentro do laço
- **d.** Lança um erro que para o programa inteiro
